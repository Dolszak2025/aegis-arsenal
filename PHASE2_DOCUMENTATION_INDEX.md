## Frequently Asked Questions

### "Where do I start as a new developer?"
**Answer:** Read [README.md](README.md) → [PHASE2_DEVELOPER_GUIDE.md](PHASE2_DEVELOPER_GUIDE.md) → Follow setup instructions. Takes ~2 hours.

### "What are my tasks for Sprint 1?"
**Answer:** Read [SPRINT1_TASKS.md](SPRINT1_TASKS.md). You'll be assigned to tasks based on your strengths. Everything you need is there.

### "How do I write database queries?"
**Answer:** See [DATABASE_QUICK_REFERENCE.md](DATABASE_QUICK_REFERENCE.md#common-queries). 20+ examples with explanations. Keep it open!

### "What's the overall architecture?"
**Answer:** [BLUEPRINT.md](BLUEPRINT.md) for high-level, [DATABASE_DESIGN.md](DATABASE_DESIGN.md) for data layer, [PHASE2_SPRINT_PLAN.md](PHASE2_SPRINT_PLAN.md) for roadmap.

### "When does Phase 2 start?"
**Answer:** April 1, 2026. See [PHASE2_LAUNCH_CHECKLIST.md](PHASE2_LAUNCH_CHECKLIST.md) for launch day procedures.

### "What's the budget?"
**Answer:** ~$189-208K for 12 weeks (2-3 engineers + DevOps + QA). See [PHASE2_READINESS_REPORT.md](PHASE2_READINESS_REPORT.md#budget-breakdown).

### "Are there code examples?"
**Answer:** Yes! [SPRINT1_TASKS.md](SPRINT1_TASKS.md) has full code examples for every task. Copy-paste ready.

### "How much time to get setup?"
**Answer:** ~4 hours total: 1 hour reading docs, 2 hours environment setup, 1 hour verification.

### "What if I find a bug?"
**Answer:** Report in #engineering on Slack. See [PHASE2_DEVELOPER_GUIDE.md](PHASE2_DEVELOPER_GUIDE.md#getting-help) for escalation.

### "Can I work on multiple sprints?"
**Answer:** Solo engineer can do 1-2 sprints. Team of 2-3 can do them in parallel. Sprints are designed to be sequential but some work can overlap.
